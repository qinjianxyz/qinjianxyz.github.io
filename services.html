How the work runs

A short path from ambiguity to evidence.

01 / Observe

See the real system

Use actual inputs, users, tools, constraints, and failure cases—not a cleaned-up description.

02 / Frame

Name the blocker

Define the owner, finish line, dependencies, unknowns, and what evidence will change the decision.

03 / Build

Make the smallest useful thing

Show a working vertical slice early and spend effort where uncertainty is highest.

04 / Verify

Prove and hand off

Test real cases, expose limitations, document operation, and make the next decision explicit.
